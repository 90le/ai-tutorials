# 旗舰内容重建设计

## 目标

删除现有 8 篇缺少研究深度、案例与视觉解释的正文，保留站点分类入口，并重建 6 篇可长期更新的旗舰教程。新内容以“AI 能力操作系统”为主线，覆盖任务定义、研究、知识管理、编程、Agent 工具与视觉生产。

## 内容组合

1. 从这里开始：AI 任务操作系统。
2. 工具教程：ChatGPT 深度研究。
3. 场景打法：AI 第二大脑。
4. 工具教程：Codex / Claude Code 智能体编程。
5. 参考资料：Agent-ready 工具设计。
6. 场景打法：AI 视觉生产。

系列课程页只负责串联这 6 篇，不复制正文。

## 编辑原则

- 公众号与 X 创作者内容用于识别读者问题、案例和讨论热点。
- 产品能力、价格、限制和安全事实优先引用官方文档、规范或一手研究。
- 每篇文章包含适合读者、前置条件、最终交付物、分步工作流、验证清单、常见失败和下一步阅读。
- 每篇文章末尾包含“参考与核验”，记录核验日期 2026-07-16。
- 不拼接、不洗写、不模仿单一作者；所有教程采用多来源交叉研究后的原创结构与表达。

## 视觉系统

- 每篇使用一张原创 SVG 章节主视觉，统一为深蓝、青绿与暖橙的编辑设计语言。
- 流程图、矩阵、检查表优先使用可访问的 HTML/Markdown/Mermaid；不把关键文字烘焙进位图。
- 官方界面截图仅在确实帮助操作时使用，并记录来源；版权不明确的公众号和 X 图片只作为研究参考，不发布。
- `public/images/flagship/` 保存最终资产，`sources.md` 记录来源与授权策略。

## 信息架构

- `start-here/`：第一次建立正确方法。
- `tools/`：围绕具体产品完成可靠任务。
- `playbooks/`：跨工具、以交付物为中心的流程。
- `series/`：学习路线与课程导航。
- `reference/`：可复用规范、模板和检查表。
- `notes/`：暂不保留薄内容，入口改为说明后续收录标准。

## 验收标准

- 8 篇旧正文不存在，旧链接不再出现在导航中。
- 6 篇旗舰正文均有完整 frontmatter、站内链接、视觉资产和来源核验。
- 分类首页准确指向新内容，没有重复导航和空泛占位。
- 静态导出下图片与链接正常。
- lint、TypeScript 和生产构建全部通过。
