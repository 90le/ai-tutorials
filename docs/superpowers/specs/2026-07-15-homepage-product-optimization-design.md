# AI Tutorials 首页产品化优化设计

## 背景与目标

现有“AI Knowledge Universe”已经建立鲜明品牌形象、六域 AI 分类和多引擎交互能力，但首页的技术展示比内容导航更突出，移动端页面偏长，实时引擎同时初始化也带来性能与维护风险。

本轮不更换视觉方向，目标是把现有实验型首页收敛为长期可用的 AI 知识产品入口：更快理解、更快进入内容、更稳定运行，同时保留足够的品牌辨识度。

## 方案比较

### 方案 A：只做性能修补

保留当前页面结构，仅延迟加载 Spline、Rive、MapLibre 等引擎。

- 优点：改动小，风险低。
- 缺点：首页仍然偏长，内容路径和可信度问题没有解决。

### 方案 B：减法式产品化优化（采用）

保留 Three.js 首屏与现有品牌视觉，压缩分类导航，前移内容和作者信息，把交互引擎降为按需进入的实验区，并统一首页与文档页令牌。

- 优点：保留品牌资产，同时改善导航效率、性能和长期扩展。
- 缺点：需要同步调整首页结构、组件边界和 CSS。

### 方案 C：完全重做为传统内容门户

删除大部分实时效果，改为资讯与文档列表首页。

- 优点：性能最好，信息密度最高。
- 缺点：丢失已经建立的独特品牌体验，不符合项目当前方向。

## 信息架构

首页按以下顺序组织：

1. 品牌首屏：一句主张、简短价值说明、两个明确入口、单一 Three.js 场景。
2. 目标入口：以“学习 AI、使用 AI、创作内容、开发产品”为四个用户目标，而不是先要求用户理解目录结构。
3. 推荐内容：真实教程、工作流和参考资料，显示内容类型与阅读目的。
4. AI 能力地图：六域分类压缩为高密度 2×3 导航，保留未来扩展能力。
5. 互动实验区：默认不加载重型引擎，由用户选择空间、状态或地图后再加载对应模块。
6. 方法路径：发现、实践、系统化三步叙事，减少垂直高度。
7. 作者与可信度：丘彬彬、微信 `binStudy`、公众号“彬彬说”、开源仓库与内容原则。
8. 页脚：完整文档分类和项目入口。

## 组件与职责

### `HomeMotionController`

- 只负责首屏入场、滚动进度、磁性按钮和轻量空间位移。
- 使用版本化 `localStorage` 保存动态开关。
- 首次访问时尊重系统 reduced-motion；用户主动选择后以用户选择为准。
- 不把正文设为透明，不控制内容是否可见。

### `KineticKnowledgeScene`

- 仍是首屏唯一自动加载的 WebGL 场景。
- 页面不可见时暂停渲染，重新可见时恢复。
- 低性能或 WebGL 初始化失败时保留 CSS 背景和完整文字入口。

### `AiEngineShowcase`

- 不再静态导入 Spline 与 Rive。
- 初始只渲染轻量选择器和说明。
- 用户选择模式后，通过动态 import 加载单个引擎。
- 切换模式时卸载上一个引擎，避免并发 WebGL 上下文和重复 Three.js 实例。
- 每个模式明确说明它与 AI 知识库的关系；通用黑色立方体场景不作为默认展示。

### 首页内容数据

- 用户目标、能力分类和推荐内容继续使用模块级静态数组。
- 所有链接指向真实站内页面，保持静态导出兼容。
- 不引入服务端 API、登录态、实时统计或虚构指标。

### 共享设计系统

- 首页和文档页共享基础背景、正文色、弱化色、边框、品牌紫和酸性绿。
- 首页允许更强的字号和动效；文档页保持较窄正文与安静表面。
- 统一按钮高度、圆角层级、焦点样式和中英文微文案规则。

## 交互与错误处理

- 首页主要导航与内容在 JavaScript 失败时仍可阅读和跳转。
- 重型引擎加载期间显示文字状态，不使用空白画布或无限加载动画。
- 引擎加载失败时显示简短说明与“重试”按钮，不影响其他首页内容。
- 动态开关固定控件不遮挡移动端正文和底部操作区域。
- 用户选择的动态状态在刷新与下次访问时保持一致。

## 性能策略

- 首屏只允许 Three.js、GSAP 与 Lenis进入自动加载路径。
- Spline、Rive、MapLibre、ECharts 和 Matter.js 按交互或接近视口后加载。
- 使用 `next/dynamic` 或原生动态 import 拆分重型客户端组件。
- 长页面非首屏区域使用 `content-visibility: auto`，但不隐藏语义内容。
- 消除重复 Three.js 警告；如果 Spline 仍不可避免引入独立 Three.js，则不与首屏场景同时挂载。

## 可访问性

- 保持语义标题顺序、真实链接、标签页语义与键盘操作。
- 为所有自定义按钮提供可见焦点样式。
- 首次访问尊重 reduced-motion；手动开启后明确显示状态。
- 画布均为增强层，核心信息必须在 HTML 中存在。
- 移动端标题限制行数和字号，避免单字断行。

## 验证与验收

1. 运行 `npm run lint`、`npm run types:check`、`npm run build`。
2. 直接检查 `out/` 静态产物。
3. 桌面与 375px 手机均满足 `scrollWidth === clientWidth`。
4. 首屏只创建一个自动 WebGL 场景。
5. 未选择实验模式时不请求 Spline、Rive 或 MapLibre 运行时代码。
6. 三个实验模式逐一加载、卸载和失败降级。
7. 控制台无重复 Three.js、RSC 预取 404 或未处理异常。
8. 首页主路径、文档导航、主题切换和作者二维码保持可用。

## 非目标

- 不重写文档内容分类。
- 不加入账户、收藏、评论、在线 AI 问答或后端服务。
- 不追求同时展示所有库；库只在能产生明确用户价值时加载。
- 不合并到 `main`，先在当前视觉升级分支提供静态预览。
