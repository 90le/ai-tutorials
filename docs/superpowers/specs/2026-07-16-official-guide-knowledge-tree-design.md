# 官方指南知识树与渲染手册设计规格

## 目标

在左侧文档目录新增顶级分类“官方指南”，把站点阅读帮助、Markdown、结构组件、代码公式表格、图示数据、图片视频、AI 交互、主题动效、无障碍和发布回归拆成多级知识树。每篇页面既是读者可学习的官方帮助，也是作者可复制的组件手册和维护者可重复执行的视觉回归样例。

## 信息架构

`content/docs/meta.json` 在首页之后加入 `help`。官方指南使用三级知识树，当前页面所在分组自动展开，其他分组默认折叠。

```text
官方指南
├─ 指南首页
├─ 阅读与导航
│  ├─ 如何使用知识库
│  └─ 搜索、目录与阅读工具
├─ 内容与排版
│  ├─ Markdown 完整样式
│  ├─ 提示、步骤、页签与折叠
│  └─ 代码、公式、表格与附件
├─ 图示与富媒体
│  ├─ Mermaid 与无限画布
│  ├─ 数据图表与可视化
│  └─ 图片、图库、视频与标注
├─ AI 交互组件
│  ├─ Agent 与工具调用
│  ├─ 提示词、上下文与检索
│  └─ 模型、成本、证据网络与 3D
└─ 设计与质量
   ├─ 主题、动画与视觉效果
   ├─ 移动端、无障碍与打印
   ├─ 组件选型指南
   └─ 完整回归检查
```

公开路径使用 `/docs/help/.../`。原 `/docs/reference/experience-lab/` 从参考资料目录移除，但保留轻量迁移页，不复制正文、不制造已有外链 404。

## 页面模板

每篇指南必须包含：适用场景、可操作演示、推荐与禁用边界、可复制 Markdown/MDX、移动与键盘行为、深浅主题、reduced-motion、静态降级、性能说明和关联页面。页面可以使用宽轨道组件，但连续正文仍走 `prose` 轨道。

## 渲染能力分层

1. Markdown 语义：H2–H6、强调、删除、标记、链接、任务项、嵌套列表、引用、分隔线、脚注、缩写、上下标、键盘键、内联和块级数学、代码与表格。
2. 结构化叙事：Callout、Cards、Steps、Tabs、Accordions、TypeTable、提示词、关键结论、场景切换、来源说明。
3. 数据与决策：指标面板、比较矩阵、数据表、公式、ECharts、证据网络和下载。
4. 媒体与空间：Mermaid、图片、图库、对比、视频、标注图和统一无限画布。
5. AI 原生交互：Agent、工具轨迹、多智能体、提示词差异、上下文预算、检索、模型权衡、成本、3D 语义、流式响应和失败恢复。
6. 质量与降级：主题、动效、移动端、键盘、屏幕阅读器、打印、无 JavaScript 和静态导出。

## 新增通用组件

在复用现有组件的前提下补齐八种常见文章表达：

- `MetricBoard`：少量关键指标、单位、变化与口径。
- `NarrativeTimeline`：按时间或阶段组织事件，可逐项聚焦。
- `ComparisonMatrix`：按维度比较方案，保留原生表格回退。
- `DecisionTreeExplorer`：分步选择并输出建议路径。
- `AnnotatedImage`：在真实图片上放置可键盘访问的标注点。
- `GlossaryGrid`：术语、定义、别名和关联页面。
- `CitationCluster`：主张、来源、日期、状态和限制。
- `RevealSequence`：用户主动推进的分镜式解释；reduced-motion 时直接显示全部。

这些组件共享 `MediaFrame`、现有颜色令牌和文章宽轨道，不引入新依赖，不复制到单篇 MDX。

## 动画与性能

- 动画只解释状态变化；默认使用 CSS transition 和用户触发，不增加自动滚动或滚动劫持。
- 所有持续动画在 reduced/off 模式关闭。
- ECharts、Three.js、Mermaid 和视频按需加载并在卸载时清理。
- 单页最多一个活动 WebGL 上下文；完整回归页不同时激活多个重型实例。
- 所有关键内容在 JavaScript 失败时仍以标题、正文、列表、表格、图片替代文本或来源信息存在。

## 图表全屏缺陷

根因是 `.docs-media-tools svg` 匹配了作为工具后代渲染的 dialog 内 ECharts SVG，把画布强制为 `0.9rem`。修复必须把图标样式限制为工具栏直接链接或按钮中的图标，并用失败测试防止回归。饼图半径和标签线同时收紧，保证窄屏全屏预览不裁切标签。
