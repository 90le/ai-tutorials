# 渲染与交互实验室设计规格

## 目标

在“参考资料”下建立公开的“渲染与交互实验室”分类，并发布一篇可长期维护的“完整渲染能力展厅”。它既向读者展示知识库能够提供的文章表现力，也作为组件、主题、响应式布局和静态导出的视觉回归页。

本次工作同时修复宽幅 Mermaid 在特定视口或浏览器缩放下过度逼近左侧导航的问题，并把预览服务收敛为唯一的 `3010` 端口。

## 已确认范围

### 范围内

- 修复 `wide` 与 `full` 媒体框在文档三栏布局中的安全宽度。
- 建立 `参考资料 / 渲染与交互实验室` 二级分类。
- 创建分类首页与“完整渲染能力展厅”公开文章。
- 展示 Markdown、Fumadocs 和项目自定义 MDX 组件的真实渲染与交互。
- 补齐当前依赖已经提供、但尚未注册到 MDX 的 Steps、Tabs、Accordions 与 TypeTable。
- 为图片、图片组、前后对比和视频创建可发布的统一素材。
- 建立能力覆盖清单、自动回归测试和桌面/移动浏览器验收矩阵。
- 关闭 `3002` 的重复静态服务，只保留 `3010`。

### 范围外

- 不把实验室升级为独立 Storybook 或组件文档应用。
- 不引入需要服务器 API、登录态或私有存储的示例。
- 不为了展示而增加无法在其他文章中复用的纯装饰组件。
- 不在示例中伪造行业统计、用户数据或产品效果。
- 不把临时生成目录、外部热链图片或不稳定远程视频发布到站点。

分类名称使用“渲染与交互实验室”，文章名称使用“完整渲染能力展厅”。`reference/meta.json` 将实验室作为独立分组加入知识树；实验室内部默认展开当前页面，并允许未来继续加入“图片与灯箱”“数据图表”“视频与字幕”等专项参考页。

分类首页负责解释用途、覆盖范围与使用边界。完整展厅负责实际渲染所有能力，不把能力清单和示例拆成互相跳转的碎片。

## Mermaid 与宽幅媒体修复

### 根因

当前 `.docs-media-frame[data-width='wide']` 和 `[data-width='full']` 使用 `calc(100% + 固定值)`、`margin-left: 50%` 与负向 `translate` 对称扩宽，再通过 `calc(100vw - 固定侧栏宽度)` 限制最大宽度。

这套算法假设左右文档栏始终占用固定宽度。实际 Fumadocs 会随视口、浏览器缩放、侧栏状态和右侧目录状态改变可用空间，因此宽幅卡片可能越过正文安全区，贴近或进入左侧导航覆盖区域。Mermaid SVG 已设置 `max-width: 100%`；问题来自外层宽幅布局，不是图形节点本身。

### 新规则

移除依赖 `100vw - 固定栏宽` 的宽度推算，改为分级媒体出血：

| 视口 | `wide` | `full` | 目的 |
| --- | --- | --- | --- |
| 小于 `90rem` | 正文宽度 `100%` | 正文宽度 `100%` | 缩放或较窄桌面不进入侧栏 |
| `90rem`–`111.99rem` | 正文两侧各扩 `2rem` | 正文两侧各扩 `3rem` | 适度提升图表可读性 |
| 大于等于 `112rem` | 最大 `62rem` | 最大 `68rem` | 真正宽屏使用宽幅展示 |

所有档位同时满足：

- 媒体框不得小于正文宽度。
- 媒体框左右边缘与可见侧栏、目录至少保留 `1.5rem` 间距。
- `.docs-media-body`、`.docs-mermaid-canvas` 和 SVG 使用 `min-width: 0` 与 `max-inline-size: 100%`。
- Mermaid 页面内视图优先完整适配；真正复杂的图使用现有全屏缩放查看器。
- 移动端不出现页面级横向滚动；源码和数据表只在自身容器内滚动。
- 打印模式统一回到 `100%` 正文宽度。

### 回归验收

浏览器检查 `1280`、`1365`、`1440`、`1638` 与 `2048` 像素宽度，并覆盖右侧目录展开与折叠状态。每个视口验证：

```text
media.left >= visibleSidebar.right + 24px
media.right <= visibleToc.left - 24px
document.scrollWidth <= document.clientWidth
```

不存在可见侧栏或目录时只验证视口安全边距与页面无横向溢出。

## 展厅叙事

展厅不采用无意义组件堆砌，而以“把一组零散 AI 调研材料变成可复核交付”为统一示例。每项控件承担真实的信息作用，读者能够理解何时使用、如何操作以及没有 JavaScript 时还能看到什么。

文章结构：

1. **如何使用这座展厅**：解释状态标记、交互方式和测试目的。
2. **基础文章语言**：标题层级、段落、强调、链接、列表、任务列表、引用、原生表格、行内代码和键盘提示。
3. **结构化说明**：Callout、Cards、Steps、Tabs、Accordions、代码标签页和 TypeTable。
4. **任务与判断组件**：PromptCard、KeyTakeaways、ScenarioTabs、SourcesDisclosure。
5. **图示与数据**：Mermaid、ECharts、公式、DataTable、CSV 下载和数据口径说明。
6. **图片与视觉证据**：ArticleImage、ImageGallery、ImageCompare、状态与来源标签、灯箱和键盘切换。
7. **视频与替代内容**：本地 MP4、封面、中文字幕、文字稿和文件回退。
8. **全局阅读系统**：章节目录、锚点、进度、返回顶部、上下篇、代码换行/展开、深浅主题和打印。
9. **降级与验收**：无 JavaScript、reduced-motion、移动端、键盘和屏幕阅读器检查方法。

## 素材系统

最终素材保存到：

```text
public/images/experience-lab/
public/videos/experience-lab/
public/downloads/experience-lab/
```

图片使用统一的 AI 知识主题视觉，包含：

- 一张“知识信号进入工作系统”的 16:9 主视觉。
- 三张连续图片，分别表达采集、判断和交付。
- 一组前后对比图：零散材料与结构化证据地图。

位图通过 ImageGen 生成，不使用 CSS 画图、临时占位图或外部热链。图片采用一致的暖白、紫色、珊瑚和酸橙点缀，避免通用蓝紫 AI 机械风。

视频为 8–12 秒的本地 MP4，以生成图片为视觉基础制作轻量镜头运动，包含：

- 1280 × 720 H.264 MP4。
- 同尺寸海报图。
- `zh-CN` WebVTT 字幕。
- 可直接阅读的 Markdown 文字稿。

视频不自动播放、不循环作为背景，不增加站点运行时渲染依赖。所有素材标注“生成内容”或“教学示意”。

下载资源包含一份证据矩阵 CSV 与一份任务合同 JSON，内容与文章中的 DataTable、PromptCard 一致。

## 组件注册与边界

`src/components/mdx.tsx` 继续作为唯一 MDX 组件注册入口。新增注册项直接从 Fumadocs 的组件模块导入，不创建重复包装层，除非现有主题无法提供必要的中文标签或无障碍语义。

展厅专用数据与示例配置放在 `src/components/docs/experience-lab/`，避免把长数组和交互逻辑塞入 MDX：

```text
src/components/docs/experience-lab/
├── showcase-data.ts
├── showcase-chart.tsx
└── showcase-types.ts
```

其中：

- `showcase-data.ts` 只保存可序列化的表格、图表和组件说明数据。
- `showcase-chart.tsx` 使用现有 `ChartFrame`，动态导入 ECharts，并在进入视口后加载。
- `showcase-types.ts` 提供 TypeTable 使用的静态接口定义。

如果 `SourceMixChart` 已足够覆盖 ECharts 的场景切换、数据表与导出能力，不再创建第二个重复图表；展厅直接复用现有组件。

## 性能与静态导出

- Mermaid、ECharts、灯箱和视频不进入首屏关键路径。
- Mermaid 与 ECharts 继续动态导入；图表进入视口后再加载。
- 图片写明宽高并延迟加载，海报优先于视频文件加载。
- 视频使用 `preload="none"`，单个 MP4 目标不超过 3 MB。
- 所有公开链接和资源在 `next build` 静态导出后存在。
- 无 JavaScript 时，核心文字、表格、源码、字幕入口与下载链接仍然存在。

1. CSS 回归测试拒绝旧的 `calc(100vw - 34rem)` 与固定对称扩宽规则。

5. 组件静态渲染测试检查状态标签、替代文本、字幕轨、表格标题和下载语义。

## 端口与预览

关闭当前 PID `37800` 所在的 `3002` 静态服务，仅保留 `3010`。后续构建仍由 `serve out --listen 3010` 提供预览；如果进程需要重启，必须先确认端口空闲，不并行启动第二个服务。

浏览器最终只保留一个 `http://localhost:3010/docs/reference/experience-lab/complete-showcase/` 交付标签页。

## 完成标准

- Mermaid 在约定视口和侧栏状态下不进入左右导航安全区。
- 页面根节点不存在横向滚动。
- `3002` 不再监听，`3010` 是唯一预览端口。
- 左侧知识树出现“渲染与交互实验室”，当前页面自动展开。
- 能力覆盖矩阵中的每一项都在文章内实际渲染或由页面级系统实际提供。
- 图片、视频、字幕、文字稿和下载文件均为本地公开资源。
- 交互同时支持鼠标和键盘，关闭弹层后焦点返回触发按钮。
- 浅色、深色、移动端、reduced-motion、打印和静态导出通过。
- `npm run check` 完整通过，浏览器控制台无 error/warning。

## 文章简报

- 主题：展示 AI 知识库支持的完整文章渲染、富媒体和交互能力。
- 目标读者：内容作者、站点维护者以及希望了解文章表现力的普通读者。
- 读者当前问题：不知道可以使用哪些组件，也无法快速判断组件在移动端、深色模式和静态站点中是否可靠。
- 完成后获得：一页可操作的能力索引、使用示例和验收基线。
- 内容类型：`reference`。
- 唯一公开路径：`/docs/reference/experience-lab/complete-showcase/`。
- 范围内：公开可复用的 Markdown、MDX、Fumadocs 与项目组件。
- 范围外：服务器能力、登录态、私有数据和第三方追踪。
- 必须核验的事实：组件注册、资源存在性、静态构建和浏览器交互。
- 图片、图表或交互及其解释价值：用真实示例解释何时使用，而不是装饰文章。
- 相邻文章与下一步阅读：实验室分类首页、Agent-ready 工具参考。

## 组件简报

- 用户价值：一页确认站点能否可靠展示目标内容形式。
- 使用页面与内容类型：实验室参考页；后续组件可复用于所有公开内容类型。
- 输入与输出：静态 MDX 数据和本地资源输入；生成可访问的 HTML、SVG、Canvas、视频和下载链接。
- 默认、加载、空、错误、禁用状态：重组件提供加载与文字回退；展示页不制造无意义空状态。
- 桌面与移动布局：桌面允许分级宽幅，移动端全部回到内容宽度。
- 键盘、焦点、屏幕阅读器：原生控件优先，弹层焦点闭环，图像/图表/视频具有名称和替代内容。
- reduced-motion 与无 JavaScript 降级：关闭非必要运动，保留正文、源码、表格、字幕入口和资源链接。
- 运行时与包体预算：重库动态导入；视频不超过 3 MB；不新增首页依赖。
- 静态导出限制：所有资源位于 `public/`，不依赖服务器 API。
- 失败测试与浏览器验收：按测试策略和完成标准执行。
