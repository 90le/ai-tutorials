# AI Tutorials Visual Upgrade Design

## Goal

将 ai-tutorials 从 Fumadocs 默认外观升级为有明确作者归属、适合中文长文阅读且能引导读者按目标学习的知识库，同时保持静态导出与现有文档链接稳定。

## Visual Direction

采用已选定的“信号与探索”方向：大面积明亮留白、近黑正文、柔和的紫—粉—橙渐变作为少量焦点，以及高对比的文本层级。渐变只用于首页英雄区和作者名片等品牌锚点；正文、导航、代码和二维码保持清晰、中性、可读。

## Homepage

首页继续保留现有的主标题和六个内容入口，但改为三段式结构：

1. 顶部英雄区说明站点价值，提供“从这里开始”和“浏览全部文档”两个入口，并使用非交互式渐变装饰建立识别度。
2. 内容入口按学习行为分组，优先展示“从这里开始、按工具学习、场景打法”，其余入口作为次级探索项。
3. 页面末尾添加“认识作者”名片，介绍丘彬彬、微信 `binStudy` 与公众号“彬彬说”。

作者名片包含两个独立的二维码触发器：

- “添加微信”对应用户提供的微信二维码图片。
- “关注公众号”对应用户提供的公众号二维码图片。
- 桌面指针设备上，悬停或键盘聚焦触发器时显示带标题和扫码说明的浮层预览。
- 触摸设备上，点击触发器后打开可关闭的模态层，显示大尺寸二维码和同样的说明。
- 所有浮层具有可访问名称、焦点管理、Escape 关闭与不依赖悬停的操作路径。
- 二维码以原始比例显示，不滤镜、不裁切、不作为背景图；资源保存为 `public/images/site/` 下有语义的文件名，并提供准确的替代文本。

## Documentation Reading Surface

保留 Fumadocs 的导航树、站内搜索、目录和 Markdown 操作。新增的主题样式仅调整阅读表面：正文最大行宽、标题间距、引用/提示块、代码区、表格、文章操作栏和链接焦点状态。文章内容结构、URL、frontmatter 与静态生成逻辑不变化。

## Information Architecture

保持当前一级目录，因为它分别覆盖学习路径、工具、连续课程、任务工作流、短内容和复用资源：

- `start-here`：新读者的入口与路线，不收纳普通文章。
- `tools`：工具使用和工具对比；每种工具使用自己的 slug 子目录。
- `series`：有顺序、最终产出的课程；每个系列保留 `index.mdx`。
- `playbooks`：以目标和场景组织的可执行工作流。
- `notes`：定位收紧为短技巧、实验和观察，导航文案改为“灵感与实验”。
- `reference`：提示词、模板、术语和 FAQ 等稳定的复用材料。

不新增一级内容目录，不迁移现有文章。后续通过 tags 提供模型、写作、编程、研究、自动化、图像、视频和效率等横向主题，文章以“入门、实战、模板、比较、复盘、实验”等内容形态帮助筛选。

## Component Boundaries

- `src/components/author-contact-card.tsx`：作者信息、二维码触发器和响应式预览行为。
- `src/components/learning-entry-card.tsx`：首页内容入口的统一卡片呈现。
- `src/app/global.css`：仅放全局 token、正文与 Fumadocs 覆盖规则。
- `src/app/(home)/page.tsx`：组合首页区块，不承载二维码交互细节。
- `src/app/docs/[[...slug]]/page.tsx`：只在确实需要结构钩子时做最小修改。

## Constraints

- 不修改 `content/docs/` 内文章正文、URL 或 frontmatter。
- 不新增后端、Cookie、登录、数据库、远程脚本或运行时私有 API。
- 不增加依赖；沿用 Tailwind、Fumadocs 与项目已有的 `lucide-react`。
- 所有变更必须通过 `npm run lint`、`npm run types:check` 和 `npm run build`。
- 需要在桌面与移动视口核对首页、作者浮层及典型文档页的可读性。
