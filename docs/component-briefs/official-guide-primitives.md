# 官方指南通用组件简报

## 用户价值

让作者面对指标、时间过程、方案比较、分支决策、图片讲解、术语、来源和分镜说明时，直接选用统一组件，而不是在每篇 MDX 中临时拼装卡片与脚本。

## 使用页面与内容类型

- 官方指南、教程、研究报告、产品介绍、案例复盘和知识参考。
- 组件：`MetricBoard`、`NarrativeTimeline`、`ComparisonMatrix`、`DecisionTreeExplorer`、`AnnotatedImage`、`GlossaryGrid`、`CitationCluster`、`RevealSequence`。

## 输入与输出

- 输入为静态、可序列化的标题、描述、数组和本地图片信息。
- 输出为语义化 `section`、列表、表格、定义列表、按钮和图片；不依赖服务端数据。

## 状态

- 默认：完整展示首个可读状态。
- 加载：仅 AnnotatedImage 的本地图片使用浏览器原生加载状态，文本标注始终可见。
- 空：显示明确空状态，不渲染空白卡片。
- 错误：无远程请求；无效输入回退为文字列表或表格。
- 禁用：已完成或不可选分支使用原生 `disabled` 与解释文字。

## 桌面与移动布局

- 桌面使用 wide 轨道和 2–4 列；移动端降为单列或组件内横向滚动。
- 不产生页面级横向滚动；触控目标不小于 40px。

## 键盘、焦点与屏幕阅读器

- 所有状态切换使用原生按钮；决策树和分镜支持方向键或 Tab 顺序。
- 当前项使用 `aria-current`、`aria-pressed` 或可读状态文字，不只依赖颜色。
- 标注图片同时输出顺序标注列表，屏幕阅读器无需坐标即可理解。

## reduced-motion 与无 JavaScript 降级

- reduced/off 关闭位移、自动推进和脉冲，仅保留短颜色过渡。
- 无 JavaScript 时首个状态、完整数据表、事件列表、术语和来源仍在服务端 HTML 中。

## 运行时与包体预算

- 不引入新依赖；使用 React 状态、Lucide 图标和 CSS。
- 单组件客户端代码目标小于 8KB gzip；无持续帧循环。

## 静态导出限制

- 不读取 Cookie、登录态、私有 API 或服务端运行时。
- 图片仅引用 `public/` 下资源；站内链接以 `/docs/` 开头。

## 失败测试与浏览器验收

- Vitest 服务端渲染覆盖八个导出名、语义结构、标签、静态回退和控件。
- 浏览器检查移动/桌面、深浅主题、键盘、reduced-motion、无横向溢出和错误浮层。
