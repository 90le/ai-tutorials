# 内容模型

文章通过 `contentType` 表明读者预期。类型统一的是内容承诺，不强制相同章节、卡片数量或验收清单。

## `tutorial`

帮助读者完成明确任务。说明前置条件、按顺序的操作、关键检查点和可验证结果。

## `explainer`

解释概念、边界和判断框架。围绕读者问题组织证据，不强制步骤结构。

## `case-study`

复盘真实或明确标注的示例。包含背景、约束、选择、结果、限制和可迁移经验。

## `reference`

提供可检索的术语、模板、对照表或接口说明。强调准确、完整和快速定位。

## `experiment`

记录假设、方法、观察、数据限制和后续验证。实验结果不得包装成普遍结论。

新文章必须选择一种类型；既有文章渐进迁移。类型不决定目录归属，文件仍按 `AGENTS.md` 的知识域保存。
