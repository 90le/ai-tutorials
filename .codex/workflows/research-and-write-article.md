# 多来源研究与写作

用于需要事实核验、热门案例比较或旗舰长文的内容任务。

1. 读取 `AGENTS.md`、`docs/content-model.md` 和相邻文章，填写 `templates/article-brief.md` 与 `templates/research-plan.md`。
2. 把问题拆成可核验子问题；关键事实优先查官方文档、论文和产品公告，社区文章只作为线索或观点。
3. 每个关键判断至少记录来源、发布日期或核验日期、原文位置、支持范围与限制；冲突来源不得静默合并。
4. 先形成独立叙事骨架，再写正文。不得复制参考文章标题、章节顺序、独特表达或未经许可的图片。
5. 只有图片、图表或交互能降低理解成本时才添加；真实数据注明口径，示意数据明确标注。
6. 完稿后抽查所有链接、数字、引用与图片权属，运行 `npm run check`。

交付物：公开 MDX、最终资产、资料与注释，以及核验日期。
