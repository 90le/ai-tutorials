这篇短教程适合遇到“回答看起来流畅，但就是不能用”的读者。你需要保留原始提示词、输入材料和不满意的回答。

完成后，你会定位主要问题，并做一次有依据的修改，而不是盲目堆叠“更专业、更详细”。

## 第一步：先给问题分类

| 症状 | 优先检查 |
| --- | --- |
| 内容跑题 | 目标和读者是否明确 |
| 编造事实 | 来源、证据和不确定性规则 |
| 太空泛 | 是否提供具体材料和使用场景 |
| 格式混乱 | 输出结构和示例 |
| 每次差异很大 | 评价标准是否可以观察 |

一次只修复最影响使用的症状。

## 第二步：检查五个层面

1. **目标**：是否写清要完成的动作，而不是只给主题。
2. **上下文**：AI 是否拿到了完成任务所需的材料。
3. **约束**：哪些内容不能猜、不能改、不能省略。
4. **证据**：结论是否需要标注来源或对应原文。
5. **输出**：格式是否适合真实使用，而不只是“详细回答”。

## 第三步：把主观评价改成可观察标准

“写得高级一点”很难验证。可以改为：

```text
开头不超过 80 字；
每个小节只回答一个问题；
专业术语第一次出现时用一句话解释；
无法从材料确认的内容标为“待核验”。
```

## 第四步：做最小修改

不要同时改写全部提示词。每次只改变一个变量，例如补充读者、增加来源规则或指定输出结构，然后比较结果差异。

## 第五步：保存失败样例

记录：

- 原提示词。
- 失败症状。
- 修改内容。
- 修改后是否改善。

这些失败样例比单独保存“万能提示词”更有长期价值。

## 验证结果

你应该能用一句话说明失败的主要原因，并指出哪一处修改改善了结果。如果只能说“换了一个更长的提示词”，说明诊断还不够具体。
