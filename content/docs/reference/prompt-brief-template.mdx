这份模板适合写作、研究、分析、开发和资料整理任务。使用前，你需要知道最终结果将交给谁、用于什么场景。

完成后，你会得到一个可以交给 AI 或协作者执行、也可以据此验收的任务说明。

## 完整模板

```text
【任务】
用一个动词说明要完成什么：整理、比较、改写、分析、实现、检查……

【使用场景】
结果将被谁在什么情况下使用？

【输入材料】
列出文件、链接、数据或原文。说明哪一份是主要依据。

【必须保留】
不可改动的事实、术语、结构或接口。

【禁止事项】
不能猜测、不能生成、不能执行或必须先确认的内容。

【输出格式】
指定章节、表格字段、代码格式、长度或文件结构。

【完成标准】
列出 3—5 条可以观察或测试的验收条件。

【不确定性处理】
遇到缺失信息时如何处理：提问、标记待确认，还是跳过。
```

## 使用步骤

1. 先填写任务、使用场景和完成标准。
2. 再补充输入材料与必须保留的内容。
3. 最后写禁止事项和不确定性处理。
4. 删除与本次任务无关的字段，避免模板本身变成噪声。
5. 让执行者先复述目标与风险，再开始生成结果。

## 精简示例

```text
任务：把访谈记录整理为产品需求摘要。
使用场景：产品经理用于确定下一轮验证问题。
输入材料：附件中的 5 份访谈记录，以受访者原话为主要依据。
禁止事项：不能把单个用户意见写成普遍结论。
输出格式：问题、出现次数、原话证据、待验证假设四列表格。
完成标准：每条结论至少关联一段原话；无法确认的频次标为“样本内”。
不确定性处理：缺失背景时列入“待确认”，不要补写。
```

## 验证结果

把简报交给不了解背景的人阅读。如果对方能说明目标、输入、风险和完成标准，就具备了可执行性。
