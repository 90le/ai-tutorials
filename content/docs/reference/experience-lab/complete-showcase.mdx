这座展厅用“把零散 AI 调研材料变成可复核交付”作为统一示例。它同时是作者参考、读者示范和站点视觉回归页。

<KeyTakeaways title="先理解三条使用原则">
  <li>先选择能够解释内容关系的组件，再考虑视觉丰富度。</li>
  <li>关键事实必须在正文、表格或替代文本中可见，不能只存在于交互里。</li>
  <li>所有图表、图片和视频都需要来源、状态与无 JavaScript 回退。</li>
</KeyTakeaways>

## 基础文章语言

正文支持**粗体判断**、*语气强调*、~~被修正的旧结论~~、`inline code`、[站内链接](/docs/)和外部链接。按 <kbd>Ctrl</kbd> + <kbd>K</kbd> 可以打开搜索。

- 无序列表用于并列信息。
  - 嵌套列表补充从属关系。
- [x] 已经核验来源。
- [ ] 仍需人工确认日期。

1. 先定义交付。
2. 再整理证据。
3. 最后统一表达。

> “看起来完整”不是验收标准。读者必须能够回到原始材料重新判断。

| 内容 | 最适合的表达 | 不应依赖 |
| --- | --- | --- |
| 顺序过程 | Steps 或有序列表 | 装饰箭头 |
| 互斥场景 | Tabs | 隐藏核心结论 |
| 可比较数据 | DataTable | 一组独立卡片 |
