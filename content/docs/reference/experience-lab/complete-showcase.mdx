## 基础文章语言

### GFM 与原生语义

正文支持**粗体判断**、*语气强调*、~~被修正的旧结论~~、`inline code`、[站内链接](/docs/)和外部链接。按 <kbd>Ctrl</kbd> + <kbd>K</kbd> 可以打开搜索。

- 无序列表用于并列信息。
  - 嵌套列表补充从属关系。
- [x] 已经核验来源。
- [ ] 仍需人工确认日期。

1. 先定义交付。
2. 再整理证据。
3. 最后统一表达。

> “看起来完整”不是验收标准。读者必须能够回到原始材料重新判断。

脚注可以把补充定义移出主叙事，同时保留回查路径。[^evidence]

[^evidence]: 这里的“证据”指能够定位到来源、日期与上下文的材料，不等同于模型生成的摘要。

| 内容 | 最适合的表达 | 不应依赖 |
| --- | --- | --- |
| 顺序过程 | Steps 或有序列表 | 装饰箭头 |
| 互斥场景 | Tabs | 隐藏核心结论 |
| 可比较数据 | DataTable | 一组独立卡片 |

## 降级与验收

关闭 JavaScript 后，正文、表格、Mermaid 源码、视频文字稿和下载入口仍应存在。减少动态偏好会关闭非必要过渡；打印时隐藏灯箱和工具栏，保留图注、数据和来源。

<ul className="docs-experience-checklist">
  <li><strong>边界：</strong>390px 到 2048px 均无页面级横向滚动，宽幅内容不进入左侧知识树。</li>
  <li><strong>输入：</strong>链接、按钮、页签、折叠、滑块和灯箱均有键盘路径与可见焦点。</li>
  <li><strong>信息：</strong>示意数字持续标注，图片有替代文本，视频有字幕和文字稿。</li>
  <li><strong>主题：</strong>浅色与深色都保持对比度，减少动态时不影响核心内容。</li>
  <li><strong>输出：</strong>静态构建、公开资源、打印和无 JavaScript 回退保持可读。</li>
</ul>

<p className="docs-experience-review"><strong>检查什么：</strong>交互增强可以消失，但文章的主张、证据、数据、来源和下一步仍然完整。</p>
