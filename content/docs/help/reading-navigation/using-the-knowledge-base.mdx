适合第一次进入站点、容易在文章间迷路的读者。完成后，你会知道三层导航分别负责什么，并能从主题入口持续读到具体实践。

<KeyTakeaways title="先记住三层位置感">
  <li>左侧是知识树：回答“我在哪个主题里”。</li>
  <li>正文是任务路径：回答“这一页要完成什么”。</li>
  <li>右侧是章节导航：回答“这一页读到哪里”。</li>
</KeyTakeaways>

## 推荐用法

<Steps>
  <Step>从左侧一级主题进入，例如“AI 应用”或“官方指南”。</Step>
  <Step>展开当前二级分组，只浏览与任务相关的页面。</Step>
  <Step>进入文章后使用右侧目录跳转；阅读完通过页尾上一篇、下一篇继续。</Step>
</Steps>

## 移动端有什么不同

移动端不会同时挤入两条侧栏。知识树和本文导航变成明确的触发入口，正文保持第一优先级。关闭抽屉后，焦点会回到触发按钮。

<Callout type="warn" title="不要用浏览器返回键代替知识结构">返回键只记录访问历史；左侧知识树才表达内容归属。跨主题阅读后，优先用知识树重新定位。</Callout>
