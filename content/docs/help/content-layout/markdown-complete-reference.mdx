这一页是基础文本语义的公开参考。它不追求装饰数量，而是确保任何教程、知识说明或参考资料都有稳定、清晰的默认表现。

## 行内语义

正文支持**重要判断**、*语气强调*、~~被更新的旧结论~~、`inline code`、[站内链接](/docs/help/)和外部链接。键盘按键使用 <kbd>Ctrl</kbd> + <kbd>K</kbd>。

原生 HTML 可以补充 Markdown 没有的语义：<mark>需要复核的重点</mark>、<abbr title="Large Language Model">LLM</abbr>、H<sub>2</sub>O 和 x<sup>2</sup>。这些标签只用于准确表达含义，不用来随意改颜色和字号。

> 好排版不会替内容做决定，但会让关系、层级和证据更容易被看见。

## 列表与任务

- 并列信息使用无序列表。
  - 从属解释可以嵌套一层。
- 不要用列表代替连续论证。

1. 先定义读者结果。
2. 再组织证据和步骤。
3. 最后选择增强组件。

- [x] 标题描述清楚
- [x] 链接可以访问
- [ ] 易变信息等待复核

## 标题层级

文章标题由页面生成，正文从二级标题开始。三级标题属于二级章节，四到六级只在确实需要表达更深从属关系时使用。

### 三级：一个明确子问题

#### 四级：子问题中的一个条件

##### 五级：少量规范细节

###### 六级：极少使用的最深层说明

## 链接、引用与脚注

关键事实应尽量链接到一手来源。补充定义可以放入脚注，避免打断主叙事。[^semantic]

[^semantic]: Markdown 首先表达语义，再由主题统一呈现；不要依靠空格和换行“画”布局。

## 可复制示例

```md title="article.md"
## 先定义任务

1. 写出读者完成后的结果。
2. 标注仍需核验的信息。[^verify]

- [x] 已确认范围
- [ ] 待补一手来源

[^verify]: 核验日期和来源要一起记录。
```
