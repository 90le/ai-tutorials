第一次接触 Agent 时，**只选一个**就够了。三个工具一起安装，会同时带来不同的登录方式、配置文件、报错提示和更新方式；你很难知道问题出在哪一个工具。先用一个获批准的 Agent 跑通“安装 → 打开 → 使用公司 Profile 连接服务”的最短路径，熟悉后再由管理员决定是否增加第二个。

本文只完成本机安装和首次启动。公司 AI 服务的地址、个人访问令牌与模型名，留到下一篇在 CC Switch 中按配置卡填写；不要提前尝试，也不要把公司 API Key 写进本文、聊天、代码、截图，或 Windows 系统/用户环境变量。

> 信息核验日期：2026-07-15。安装命令和支持平台会变化，请以各工具的官方一手文档为准：[Codex CLI 安装说明](https://github.com/openai/codex#installing-and-running-codex-cli)、[Claude Code 安装说明](https://code.claude.com/docs/en/setup)、[OpenCode 安装说明](https://opencode.ai/docs/)。

## 适合谁

- 使用 Windows 10/11，第一次接触终端型 AI Agent 的业务、研究、营销或数据分析同事。
- 已获管理员批准使用 CC Switch 和至少一个 Agent，但还没有配置公司 AI 服务的员工。
- 希望先完成一个可控、可求助的最小安装，而不是同时比较多个工具的人。

## 前置条件

- 已完成[开始前：账号、数据与安全边界](./01-before-you-start/)，并持有管理员发放的配置卡或知道支持渠道。
- 使用公司批准的电脑、网络和本地账户；如软件安装受管控，请按公司的软件申请流程操作。
- 准备一个空的本地测试文件夹，不要把客户资料、生产数据或公司项目仓库作为第一次启动位置。
- 只从下方的官方链接或公司软件中心安装。不要从群文件、网盘或搜索广告下载安装包。

## CC Switch 在这里负责什么

CC Switch 是公司用来管理获批准 AI 服务配置的入口，**不是让你同时学习多个 Agent 的理由**。在后续配置中，你会在 CC Switch 选择管理员提供的 Profile，并把它用于已验证的 Agent 组合。

不同 Agent 的配置字段与协议并不相同。公司网关也未必支持每一种 Agent 或每一种协议；只有管理员明确写在配置卡上的“Profile + Agent”组合才可使用。本文的“安装成功”不等于已经连上公司模型，也不表示可以自行填写第三方 Provider、兼容接口或 API Key。

## 先做选择：三选一

| 选项 | 适合的起点 | Windows 提醒 | 官方来源 |
| --- | --- | --- | --- |
| Codex CLI | 管理员批准使用 Codex，或你想从一个简洁的终端入口开始 | 官方提供 PowerShell 安装方式 | [OpenAI / openai/codex](https://github.com/openai/codex#installing-and-running-codex-cli) |
| Claude Code | 管理员批准使用 Claude Code，且你希望使用 Windows 原生安装器 | 原生 Windows 可用；官方也说明 WSL 2 适合需要 Linux 工具链的场景 | [Anthropic 官方安装文档](https://code.claude.com/docs/en/setup) |
| OpenCode | 管理员已明确验证 OpenCode 与公司 Profile 的兼容性 | 官方建议 Windows 优先使用 WSL；原生 Windows 安装可作为管理员批准的例外路径 | [OpenCode 官方文档](https://opencode.ai/docs/) |

选择管理员推荐的那个；如果配置卡没有写明，先问管理员“我应使用哪一个 Agent”，不要靠价格、模型列表或同事偏好猜测。下面三条路径中只执行**一条**。

## 路径 A：安装并启动 Codex CLI

1. 在 Windows 开始菜单搜索并打开 **PowerShell**；不要在命令中加入公司令牌，也不要先设置任何 `OPENAI_API_KEY` 或其他环境变量。
2. 按 [Codex CLI 的 Windows 官方安装说明](https://github.com/openai/codex#installing-and-running-codex-cli)运行安装命令：

   ```powershell
   powershell -ExecutionPolicy ByPass -c "irm https://chatgpt.com/codex/install.ps1 | iex"
   ```

3. 关闭 PowerShell，再重新打开一个 PowerShell 窗口，确认命令已可用：

   ```powershell
   codex --version
   ```

4. 进入你的空测试文件夹后启动：

   ```powershell
   cd $HOME
   mkdir agent-first-run -ErrorAction SilentlyContinue
   cd agent-first-run
   codex
   ```

5. 首次界面如要求登录，只使用管理员允许的登录方式。若界面要求填写 API Key、服务地址或模型，先退出；这些值应在下一篇通过 CC Switch 的公司 Profile 配置，而不是写入环境变量或本地命令。

## 路径 B：安装并启动 Claude Code

1. 在开始菜单打开 **PowerShell**。Claude Code 的官方文档说明原生 Windows 可用，且不需要以管理员身份运行安装命令。
2. 按 [Claude Code 官方安装说明](https://code.claude.com/docs/en/setup)运行 Windows PowerShell 安装命令：

   ```powershell
   irm https://claude.ai/install.ps1 | iex
   ```

3. 关闭并重新打开 PowerShell，确认安装成功：

   ```powershell
   claude --version
   ```

4. 在空测试文件夹中启动：

   ```powershell
   cd $HOME
   mkdir agent-first-run -ErrorAction SilentlyContinue
   cd agent-first-run
   claude
   ```

5. 按界面完成公司批准的首次登录即可。若出现个人订阅、账单、API Key、Provider 或环境变量设置，请不要继续填写，记录提示文字后退出并联系管理员。

## 路径 C：安装并启动 OpenCode

1. 只有在管理员确认“公司 Profile 已验证支持 OpenCode”时才选择本路径。OpenCode 的官方文档建议 Windows 用户优先使用 WSL；如果管理员没有给出 WSL 方案，不要自行切换或安装未知环境。
2. 若你的电脑已经按公司标准安装 Node.js 和 npm，请在 PowerShell 中按 [OpenCode 官方 Windows 安装说明](https://opencode.ai/docs/)运行：

   ```powershell
   npm install -g opencode-ai
   ```

   如果公司要求 Chocolatey 或 WSL，请只使用管理员给出的受控安装路径，不要混用多种安装方式。

3. 关闭并重新打开 PowerShell，确认命令可用：

   ```powershell
   opencode --version
   ```

4. 在空测试文件夹中打开 OpenCode：

   ```powershell
   cd $HOME
   mkdir agent-first-run -ErrorAction SilentlyContinue
   cd agent-first-run
   opencode
   ```

5. 暂时不要执行 `/connect`，也不要按引导粘贴个人或公司 API Key。先让管理员确认 CC Switch Profile 所使用的 Provider 与协议是否受 OpenCode 支持；未验证时，能打开界面就是本篇的完成标准。

## 验收：你现在应看到什么

完成任一路径后，逐项检查：

- 你只安装了一个 Agent，并能在新开的 PowerShell 中运行对应的 `--version` 命令。
- Agent 能在 `agent-first-run` 这个空文件夹中打开，且没有读取客户文件或执行任务。
- 没有在 Windows 环境变量、终端历史、配置文件、截图或聊天中写入公司 API Key、服务地址、访问令牌或模型白名单。
- 你知道自己选择的 Agent 名称，并知道它是否已被管理员列为 CC Switch Profile 的已验证组合。

满足前两项但还不能对话是正常的：公司服务尚未在 CC Switch 中配置。不要为了“先试一下”而改用个人 Key 或公开中转服务。

## 自助排错

### `codex`、`claude` 或 `opencode` 显示“不是内部或外部命令”

先关闭所有 PowerShell 窗口并重新打开，再运行一次对应的 `--version` 命令。仍失败时，把命令名称和完整报错文本发给支持渠道；不要在网上复制修改 PATH、注册表或安全策略的命令。

### 安装命令提示语法错误，或下载被拦截

先核对你使用的是 **PowerShell** 还是 CMD；Claude Code 的官方安装页专门区分两种命令。企业网络、安全软件或软件中心策略也可能阻止下载。请把错误信息交给 IT/管理员，由他们提供受控安装方式；不要关闭防护软件，也不要使用来历不明的镜像。

### 界面要求 API Key、付款信息或选择第三方 Provider

停止在该界面继续操作。本文不使用公司 API Key，也不设置系统环境变量；是否支持该 Agent、Provider 或协议必须由管理员确认。把产品名称、页面提示和截图（先遮盖账号信息）发给支持渠道。

### CC Switch 中没有我的 Agent，或没有可选的公司 Profile

这通常表示该 Agent 组合尚未验证、你没有权限，或管理员尚未发放配置。不要手工创建“兼容 OpenAI”之类的猜测配置。请提供你选择的 Agent 名称、CC Switch 版本和看到的提示，申请已验证的配置卡。
