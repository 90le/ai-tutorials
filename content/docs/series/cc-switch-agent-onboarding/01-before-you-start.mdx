AI Agent 能读文件、调用工具，有些还能执行命令。先确认边界，比先安装更多工具更重要。

## 适合谁

第一次使用 CC Switch、Codex CLI、Claude Code、OpenCode 或其他 Agent 的 Windows 员工。

## 你需要准备

- 公司批准使用的电脑与网络环境。
- 公司管理员或培训负责人提供的 AI 服务配置卡。
- 一个用于测试的非敏感示例文件或公开材料。

不要使用客户原始数据、未发布方案、个人信息、访问令牌或生产数据进行第一次测试。

## 向管理员索取什么

配置卡应通过公司的受控渠道发放，并至少说明：

| 项目 | 用途 |
| --- | --- |
| 推荐的 CC Switch Profile 名称 | 帮你选择正确的公司配置，而不是猜测 Provider。 |
| 可使用的 Agent | 例如先使用一个获批准的 CLI Agent。 |
| 服务地址与模型名 | 仅在 CC Switch 的对应配置中填写；不要转发给无权限人员。 |
| 个人访问令牌的领取/轮换方式 | 每人独立、可撤销；不要共享群聊里的固定 Key。 |
| 数据范围与支持渠道 | 明确哪些资料可处理，遇到问题找谁。 |

## 三条底线

1. **不共享密钥。** 不把令牌粘贴到文档、截图、聊天群、代码仓库或系统环境变量中。
2. **不上传未经批准的数据。** 不确定数据是否可用时，先询问项目负责人或数据安全负责人。
3. **不让 Agent 自动执行高风险动作。** 第一次使用时，只让它解释、整理或生成草稿；涉及删除、发送、发布、运行脚本和写入生产系统时必须人工确认。

## 第一次练习

选择一段公开新闻、公开产品介绍或你自己编写的无敏感测试文本，完成一个低风险任务：

```text
请把下面内容整理成 5 条要点，并标出其中需要进一步核实的说法。
```

验收标准：输出可读、没有泄露配置值，且你能指出哪些结论需要人工核验。
