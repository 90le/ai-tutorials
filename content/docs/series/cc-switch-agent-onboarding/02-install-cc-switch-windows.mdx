本节带你在 Windows 电脑上安装公司已发放的 CC Switch 安装包，并完成首次启动和运行环境自检。完成后，你会知道该向管理员索取哪些个人配置，而不会把公司共用的上游密钥写进电脑或文档。

## 适合谁

- 第一次使用公司 AI Agent 工具的 Windows 员工。
- 已收到公司 IT、管理员或培训负责人发放的 CC Switch 安装包，但尚未配置个人访问权限的同事。

## 开始前准备

1. 使用 Windows 10 或更高版本的 x64 电脑。CC Switch 官方安装指南将 Windows 10 及以上、x64 列为 Windows 支持条件。[核验日期：2026-07-15；官方安装指南](https://github.com/farion1231/cc-switch/blob/main/docs/user-manual/zh/1-getting-started/1.2-installation.md)
2. 准备公司发放的安装包；不要自行从聊天记录、网盘转载或不明网站下载替代包。CC Switch 官方指定渠道为 [ccswitch.io](https://ccswitch.io/zh/) 和 [GitHub Releases](https://github.com/farion1231/cc-switch/releases)。
3. 如电脑受公司设备管理，请确认你有安装应用的权限；没有权限时联系 IT，不要尝试绕过公司的安装限制。
4. 暂时不需要填写任何公司上游总 Key、共享账号密码或管理员密钥。

## 先认识三个词

为便于团队协作，本系列采用以下说法：

| 词语 | 在本系列中的含义 | 你需要做什么 |
| --- | --- | --- |
| Provider（供应商） | 一组让某个 AI 工具连接到服务的配置。官方文档说明 CC Switch 可管理 Claude Code、Codex、Gemini CLI 等工具的供应商配置。 | 不要自行猜测端点或密钥。 |
| Profile（配置档） | 公司为某个使用场景准备的推荐配置组合，例如指定 Agent、Provider、模型和使用规则。它是团队约定，不等同于公开文档中的密钥。 | 向管理员索取推荐的 Profile 名称或导入方式。 |
| Agent（智能体工具） | 你实际打开来完成任务的 AI 工具；CC Switch 可在一个桌面应用中管理多种受支持工具。 | 只选择管理员指定的 Agent，不随意改用其他工具。 |

CC Switch 的官方仓库说明，它以桌面应用统一管理受支持工具的配置，并提供供应商切换能力；不同工具的配置格式并不相同。[官方说明](https://github.com/farion1231/cc-switch)

## 安装与首次启动

1. 在公司发放位置找到安装包，确认文件名、发放人和接收渠道与公司通知一致。公司若同时提供校验值或签名说明，按该说明核对；不确定就暂停并询问管理员。
2. 双击安装包并按 Windows 安装向导完成安装。CC Switch 官方 Windows 文档列出了安装包安装流程；本页不固定具体版本号或按钮文字，以你收到的公司包和屏幕提示为准。
3. 安装完成后，从 Windows 开始菜单或公司软件门户启动 CC Switch。首次打开后，确认主窗口正常显示；官方安装指南还将系统托盘出现 CC Switch 图标列为安装验证信号之一。
4. 在应用中选择管理员要求使用的 Agent 面板。不要因为看到其他工具就额外安装、登录或配置它们。
5. 按公司首装说明运行 **运行环境自检**。不同发布版本的入口和文案可能变化，因此以公司指引和应用实际显示为准，不根据本页猜测菜单名称。自检应至少确认：目标 Agent 已被识别或已具备运行条件；网络、公司访问令牌和配置格式没有报错。
6. 如果管理员提供了可导入的公司 Profile，先阅读导入预览，确认它只对应你的个人访问权限和指定 Agent；再完成导入。CC Switch 官方文档说明它支持通过 `ccswitch://` 深度链接导入配置，导入时会显示确认步骤。[官方用户手册](https://github.com/farion1231/cc-switch/blob/main/docs/user-manual/zh/2-providers/2.1-add.md)
7. 如管理员要求手动填写，请只填写管理员单独、安全地给你的四项信息：**公司访问令牌 + Base URL + 模型名 + 推荐 Profile**。逐字复制，不要自行补全 URL 路径、替换模型名或把令牌发到群聊。
8. 保存并启用管理员指定的配置。官方文档指出，切换供应商后多数工具需要重启终端或对应 CLI 才会读取新配置；是否需要重启以当前 Agent 的提示和公司说明为准。[官方快速上手](https://github.com/farion1231/cc-switch/blob/main/docs/user-manual/zh/1-getting-started/1.4-quickstart.md)

## 绝不输入公司上游总 Key

公司上游总 Key 是管理员用于服务端、网关或集中管理的高权限密钥，不是员工客户端的登录凭据。它不应出现在 CC Switch 表单、截图、聊天记录、工单正文或本教程中。

如果某人只给了你「总 Key」而没有给个人访问令牌和推荐 Profile，请停止配置并向管理员说明：**我需要公司访问令牌、Base URL、模型名和推荐 Profile；我不会使用上游总 Key。**

同样地，看到“付费、充值或索取登录凭据”的非公司 CC Switch 下载页或客户端时，不要继续操作。官方安装指南明确提示仅使用官网、官方 GitHub Releases 或源码仓库获取软件。

## 验证结果

完成本节后，逐项确认：

- CC Switch 能在 Windows 正常打开，且可在系统托盘看到图标。
- 运行环境自检没有阻断性错误；若有错误，已记录错误提示并交给管理员或 IT。
- 你已选定一个公司指定的 Agent 和推荐 Profile。
- 你只使用了发给自己的公司访问令牌，未输入、复制或保存公司上游总 Key。
- 启动指定 Agent 后，按公司提供的无敏感数据测试任务得到正常响应；如该 Agent 要求重启，已重启后再验证。

## 常见问题

### 安装包打不开，或 Windows 不允许安装

先确认包来自公司发放渠道，并把 Windows 提示的完整文字或截图发给 IT。不要下载“破解器”、关闭公司安全策略，或换用来源不明的安装包。

### 自检提示找不到 Agent

把自检结果、你选择的 Agent 名称和电脑环境信息交给管理员。管理员会确认该 Agent 是否需要由公司软件门户安装，或是否需要补充运行环境；不要自行安装来历不明的命令行工具。

### 配置后没有生效

确认已启用管理员指定的 Profile，并按提示完全退出后重开目标 Agent 或终端。官方文档说明多数工具需要重启才会读取切换后的配置；仍未生效时，把报错信息交给管理员，不要随意修改 Base URL、模型名或协议选项。

### 我需要 API Key 吗？

使用公司环境时，请求管理员发放**公司访问令牌**及对应的 Base URL、模型名和推荐 Profile。不要索取、共享或代填公司上游总 Key；如果公司采用其他个人凭据形式，也只通过管理员指定的安全渠道接收。
