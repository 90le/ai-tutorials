这一篇只讲安全的配置流程，不提供公司网关地址、真实模型名、令牌或内部截图。请以管理员发放的配置卡为唯一准则。

## 前置条件

- 已完成 [Windows 安装 CC Switch](./02-install-cc-switch-windows/)。
- 已安装并初始化至少一个获批准的 Agent。
- 已取得个人配置卡；若没有，请先联系管理员，不要自行尝试公开中转服务或向同事索要 Key。

## 为什么要用配置卡

公司服务应为每位员工提供可撤销、可追溯的个人访问方式。这样管理员可以：

- 按部门和角色限制可用模型与能力；
- 汇总用量、控制成本、发现异常；
- 在员工离岗、令牌泄露或配置升级时单独撤销或更新；
- 避免把上游模型厂商的总 Key 放到每台电脑上。

## 配置流程

1. 打开 CC Switch，确认正在编辑的是公司批准的 Profile，而不是个人实验 Profile。
2. 根据配置卡填写服务地址、个人访问令牌和指定模型名。只填写管理员提供的值，不要根据网络文章猜测。
3. 将 Profile 绑定到你选择的 Agent。不同 Agent 的协议和配置字段可能不同；公司只应提供已验证的组合。
4. 保存后关闭并重新打开目标 Agent，使其读取最新配置。
5. 用无敏感内容执行一次最小测试，例如“用一句话解释什么是市场细分”。

## 验证结果

满足以下条件即表示基础连接成功：

- Agent 正常返回结果；
- CC Switch 中显示的是公司批准的 Profile；
- 测试过程中没有出现令牌、内部服务地址或客户资料；
- 你知道出现错误时应联系的管理员或支持渠道。

## 常见问题

### 配置已保存，但 Agent 仍然使用旧模型

关闭并重新启动 Agent，再确认本机没有残留的 API 环境变量。环境变量可能覆盖 CC Switch 写入的配置；不要自行删除不认识的公司环境变量，应把现象和截图发给管理员。

### 我能把配置卡转给同事吗

不能。配置卡中的令牌和服务参数只属于你本人。请同事通过正式流程领取自己的配置。

### 我想接入新的模型或第三方插件

先提交需求给管理员。新模型、Skill、MCP 或插件应经过兼容性、安全和成本评估后再加入公司 Profile。
