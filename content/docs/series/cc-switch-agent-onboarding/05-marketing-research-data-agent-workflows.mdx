这篇适合营销、市场研究、媒介策略和数据分析团队中需要日常处理公开资料、访谈纪要或已脱敏数据的员工。完成后，你能够按场景选择企业批准的 Agent/Skill 包，写出边界清晰的任务提示词，并在提交成果前完成必要的人工复核。

## 开始前：明确什么可以交给 Agent

先确认以下前置条件：

- 你已获得企业批准的 CC Switch 使用方式和 Provider 列表。
- 管理员已审核、安装并版本化本教程涉及的 Agent/Skill 包；员工不自行从未知来源安装或更新。
- 任务所需资料已完成最小化处理：只保留完成任务所必需的字段、片段和时间范围。
- 你知道团队的数据分级规则，以及需要升级给负责人或法务/安全团队的情形。

<Callout type="warn" title="先核对 Provider，再粘贴内容">
不得把客户资料、个人信息、未公开投放数据、账号凭据、内部策略、源数据导出文件或其他敏感数据发送给未经企业批准的 Provider。即使 Agent 表现为“只做摘要”，也必须先确认它实际调用的模型和数据处理路径都在批准范围内。
</Callout>

这里的“精选 Agent/Skill 包”不是员工个人的工具收藏夹，而是一套由企业管理的能力目录：管理员负责来源审核、权限评估、版本号、变更记录和下线机制；员工只从目录中按场景选用，并在版本变更后重新阅读说明。这样能降低提示词注入、越权访问、不可复现结果和第三方数据外流的风险。

## 通用操作流程

每次运行前后，按下面流程操作。

1. 在企业目录中选择匹配场景、当前已批准版本的 Agent/Skill 包；阅读它的用途、输入限制、可调用 Provider 与输出格式说明。
2. 只准备完成当前任务所需的公开资料、已授权资料或经脱敏的摘要。删除姓名、手机号、邮箱、设备标识、客户名称、精确预算、订单明细等不必要字段。
3. 在提示词中写清任务目标、允许使用的材料、禁止推断的内容、期望格式和不确定时的处理方式。不要要求 Agent 绕过规则、访问外部账号或自行上传文件。
4. 先用一小段低敏感样本试运行；检查 Agent 是否遵守范围、是否引用了不存在的材料、是否把假设写成结论。
5. 人工复核输出的事实、计算、引文、品牌表述和建议，再将结果写入正式报告。保留任务版本、输入来源类别、运行日期和复核人，便于追溯。

## 场景一：从公开材料提炼营销洞察

**适用任务**：整理公开活动页面、公开社媒评论样本或已获授权的访谈摘要，形成待验证的受众洞察假设。

**推荐包能力**：公开网页/文本整理、主题聚类、结构化写作。它不应要求连接客户 CRM、广告账户或个人社交账号。

**可复制提示词框架**：

```text
你是营销洞察助理。只基于我提供的【公开或已授权材料】工作，
不要访问链接以外的账号、系统或文件，也不要补充未提供的事实。

任务目标：围绕【产品/品类】归纳受众关注点，生成供人工验证的洞察假设。
材料范围：
- 来源类型与日期：【例如：公开活动页，2026-06】
- 已脱敏材料：【粘贴必要片段】

请输出：
1. 3—5 个主题，每个主题附对应材料片段或编号；
2. 每个主题的“观察”与“待验证假设”，必须明确区分；
3. 缺失信息、样本偏差和不能下结论的地方；
4. 不要给出个体画像、敏感属性推断或投放决策。
```

**人工复核重点**：检查结论是否把少量评论当作整体市场；删除可能识别个人的原句；将“用户都认为”改为与样本相称的表述，例如“在提供的样本中反复出现”。

## 场景二：把研究材料整理为可核查摘要

**适用任务**：将公开报告、已获授权的访谈纪要或会议记录压缩成研究摘要，方便研究员继续核验。

**推荐包能力**：长文本结构化、引文定位、矛盾点提示。Agent 应将材料中已有的信息与自己的推断分开呈现。

**可复制提示词框架**：

```text
你是研究摘要助理。仅依据下列材料生成内部初稿；材料未说明的内容请标记“未提供”，不要猜测。

研究问题：【填写一个可回答的问题】
材料说明：【公开/已授权/已脱敏；来源日期与范围】
材料内容：【粘贴必要片段，使用 [S1]、[S2] 等编号】

请按以下结构输出：
- 一句话摘要（不超过 60 字）；
- 已有证据：每点注明 [S1] 等来源编号；
- 不一致或证据不足之处；
- 可由研究员进一步验证的 3 个问题；
- 明确列出假设，不把假设写成事实。

不要输出个人身份信息、客户名称或材料之外的引文。
```

**人工复核重点**：逐条打开或核对来源编号，确认引文没有被截断而改变含义；对包含研究参与者信息的材料，优先传递去标识化摘录，而非完整原始记录。

## 场景三：解释已脱敏的汇总数据

**适用任务**：检查已聚合、已脱敏的数据表，生成异常说明、可视化建议或分析检查清单。

**推荐包能力**：表格理解、统计说明、代码/公式草拟。它不应直接获得生产数据库连接、全量明细或含唯一标识符的文件。

**可复制提示词框架**：

```text
你是数据分析协作助手。只分析下面的已脱敏、已聚合数据；
不得尝试识别个人、企业或客户，也不得要求更多原始明细。

业务问题：【例如：哪些渠道的周度指标值得复核？】
字段说明：【字段名、口径、时间范围、缺失值含义】
数据样本：【仅粘贴完成任务所需的汇总行】

请输出：
1. 数据质量检查清单（缺失、重复、口径变化、异常值）；
2. 描述性发现，区分“数据直接显示”与“需要验证的解释”；
3. 建议进一步计算的指标或图表，并写明所需口径；
4. 不确定性和可能的混杂因素。

不要给出自动化投放、预算调整或归因结论；不要把相关性描述为因果关系。
```

**人工复核重点**：由数据负责人确认字段口径、聚合粒度、样本量和统计方法；在对外材料中只保留已经过业务与合规审核的图表和表述。

## 什么时候停止并升级处理

出现以下任一情况时，停止运行、不要继续补充数据，并联系团队负责人或安全/合规负责人：

- 不确定某个 Provider、Agent、Skill 版本或连接器是否已获批准；
- 输入包含个人信息、客户保密信息、未公开业务指标、账号凭据或受合同限制的第三方资料；
- Agent 要求关闭安全限制、上传原始文件、粘贴令牌，或访问未授权的系统；
- 输出看似包含未提供的数据、对个人作出敏感推断，或给出高影响决策建议；
- 你无法说明一项结论来自哪份材料、哪次运行和哪个版本。

## 交付前检查清单

- [ ] 使用的是企业目录中已批准、当前版本的 Agent/Skill 包和 Provider。
- [ ] 输入只包含完成任务必需的公开、已授权或已脱敏信息。
- [ ] 输出中的事实、数字、引文和计算均已由人工复核。
- [ ] 假设、观察和结论已明确区分；不确定性没有被隐藏。
- [ ] 已删除不必要的个人信息、客户信息、凭据和内部数据。
- [ ] 已记录所用包版本、材料来源类别、运行日期与复核责任人。
